# Сканер ROM-адресов DS18b20

Отдельный мини-проект PlatformIO, **не часть основной прошивки** `garage-heat-v2`.
Нужен один раз — чтобы снять реальные ROM-адреса трёх датчиков DS18b20 на шине
D3 и правильно сопоставить их с ролями BLOWN_AIR/TARGET/INFO (CLAUDE.md §2/§7)
перед тем как прописать их в `include/config.h` основной прошивки.

Заодно скетч проверяет шины D2 (уличный воздух) и D4 (радиатор SSR) — там
должно быть ровно по одному датчику; если находится 0 или больше 1 — это
ошибка проводки, а не адресации.

## Почему это нужно

На шине D3 физически три одинаковых датчика на одном проводе. Прошивка
различает их не по порядку подключения, а по зашитому в `config.h`
64-битному ROM-коду (`ROM_BLOWN_AIR`/`ROM_TARGET`/`ROM_INFO`). Пока эти
ROM-коды не сняты с реального железа, в `config.h` лежат заглушки — прошивка
с ними не имеет смысла для реальной работы.

## Как прошить и снять адреса

Из этого каталога (не из корня основного проекта):

```bash
cd tools/scan_ds18b20
pio run -t upload
pio device monitor -b 115200
```

При старте скетч печатает для каждой шины количество найденных устройств и
их ROM-адреса сразу в формате, готовом для вставки в `config.h`, например:

```
D3 (BLOWN_AIR/TARGET/INFO): найдено устройств = 3
  #0 ROM = { 0x28, 0x1A, 0x2B, 0x3C, 0x4D, 0x5E, 0x6F, 0x01 }
  #1 ROM = { 0x28, 0x99, 0x88, 0x77, 0x66, 0x55, 0x44, 0x02 }
  #2 ROM = { 0x28, 0x11, 0x22, 0x33, 0x44, 0x55, 0x66, 0x03 }
```

Если `найдено устройств` не совпадает с ожидаемым (3 для D3, 1 для D2/D4) —
проверь провод и подтяжку 4.7 кОм между DQ и VCC на соответствующей шине,
прежде чем читать дальше.

## Как сопоставить ROM с ролью (BLOWN_AIR/TARGET/INFO)

Проблема: сканер видит три ROM-адреса на D3, но не знает, какой физический
датчик (провод) за каким адресом стоит. Два способа это выяснить —
используй тот, что удобнее по месту:

### Способ А — по одному датчику на шине (надёжнее, рекомендуется)

Пока датчики ещё не смонтированы окончательно:

1. Подключи к D3 **только один** физический датчик (два других отключи).
2. Прошей/перезапусти монитор — в блоке `D3 (...)` при старте будет
   `найдено устройств = 1` и один ROM. Запиши его и подпиши сам датчик
   (например, малярным скотчем: "1", "2", "3").
3. Повтори для двух оставшихся датчиков по отдельности.
4. В конце у тебя есть однозначная таблица: физический датчик → ROM.
5. Смонтируй датчики по их назначению (CLAUDE.md §2):
   - датчик в потоке горячего воздуха от нагревательного элемента → BLOWN_AIR
   - датчик в зоне хранения (где мотоцикл) → TARGET
   - оставшийся, куда угодно ещё → INFO
6. Подключи все три вместе на D3 и один раз прошей финальную прошивку с
   готовыми ROM в `config.h`, чтобы убедиться, что `найдено устройств = 3`.

### Способ Б — по нагреву на месте (если датчики уже смонтированы)

Если датчики уже приклеены/закреплены на местах и переставлять их сложнее,
чем прогреть на месте:

1. Подключи все три датчика на D3 сразу, прошей, открой монитор — в `loop()`
   каждую секунду печатаются live-показания всех трёх `D3 #0/#1/#2` с ROM.
2. Поочерёдно грей (пальцем, феном на минимальной мощности, издалека) или
   остужай (лёд, спрей) один физический датчик за раз и смотри, у какого `#`
   в мониторе меняется температура быстрее и заметнее остальных.
3. Так сопоставляешь физическое место установки (а значит и роль по
   CLAUDE.md §2 — BLOWN_AIR/TARGET/INFO) с конкретным `#` и его ROM.

## После того как адреса известны

Открой `include/config.h` основной прошивки и замени заглушки на реальные
значения (порядок ролей — по смыслу, не по индексу `#`):

```cpp
#define ROM_BLOWN_AIR { 0x28, 0x1A, 0x2B, 0x3C, 0x4D, 0x5E, 0x6F, 0x01 }
#define ROM_TARGET    { 0x28, 0x99, 0x88, 0x77, 0x66, 0x55, 0x44, 0x02 }
#define ROM_INFO      { 0x28, 0x11, 0x22, 0x33, 0x44, 0x55, 0x66, 0x03 }
```

Заодно убери комментарий "ЗАГЛУШКИ" над этим блоком в `config.h` — он больше
не актуален.

Это разовая правка `config.h` перед первой реальной прошивкой на железо;
`tools/scan_ds18b20` после этого больше не нужен для обычной работы —
это отдельный проект, он не собирается и не мешает основной прошивке.
